## Key Takeaways (Final Principles)
1. Cloud native is a **development methodology**, not just a technology choice
2. Use the **four facilities** (Lab/Factory/Citadel/Observatory) as your mental model
3. Start simple; **resist premature complexity** (Cloud Run → GKE only when needed)
4. **Immutable artifacts** flow through environments unchanged
5. **Automation eliminates toil** — if you do it twice, automate it
6. **Observability is designed**, not added retrospectively
7. **Security is shared responsibility** — built in at every layer, not bolted on
8. **Open standards** give portability without sacrificing managed service benefits
9. Developer joy is a legitimate engineering goal — friction-free environments produce better software
