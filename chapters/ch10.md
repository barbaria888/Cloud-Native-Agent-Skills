# Ch10 — Building the Laboratory

## The Inner Loop
```
Write Code → Build Container → Deploy to Local K8s → Test → Repeat
```
Goal: make this loop as fast as possible (seconds, not minutes).

## Development Environment Options

| Environment | Cost | Local Overhead | Suitable For |
|---|---|---|---|
| Local + Local K8s (Minikube) | Low | Yes | Full control, offline dev |
| Cloud Shell + Cloud Code | Free | No | Zero-setup, always current |
| Cloud Workstations | Paid | No | Team standardization, powerful |

**Cloud Shell Editor tip:** Comes with 2 vCPU / 4GB Minikube included, 5GB persistent disk — zero-cost K8s environment.

## Skaffold (The Inner Loop Tool)
```bash
# Initialize (auto-detects build tool)
skaffold init --generate-manifests

# Build container once
skaffold build

# Deploy once to K8s
skaffold run --port-forward

# Watch mode: rebuild + redeploy on file save
skaffold dev

# Debug mode with breakpoints
skaffold debug
```

**skaffold.yaml example (Go with Ko):**
```yaml
apiVersion: skaffold/v4beta5
kind: Config
metadata:
  name: skill-service
build:
  artifacts:
    - image: skill-service
      ko:
        baseImage: gcr.io/distroless/static:nonroot
        importPath: github.com/SkillsMapper/skillsmapper/skill-service
manifests:
  rawYaml:
    - k8s/deployment.yaml
```

**skaffold.yaml example (Java with Jib):**
```yaml
build:
  artifacts:
    - image: fact-service
      jib:
        project: org.skillsmapper:fact-service
```

## Container Build Tools Comparison
| Tool | Language | Dockerfile Needed | Speed |
|---|---|---|---|
| Cloud Native Buildpacks | Any | No | Slow (first run) |
| Jib | JVM | No | Fast (incremental) |
| Ko | Go | No | Very fast |
| Docker | Any | Yes | Variable |

**Rule:** Use the fastest tool for your language in the inner loop.

## Local K8s Options
- **Minikube**: Provided in Cloud Shell; good default
- **Docker Desktop K8s**: Easy on Mac/Windows; single-node
- **kind (K8s in Docker)**: Lightweight; good for CI testing

## Kubernetes Manifests (Minimal Deployment)
```yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: skill-service
spec:
  replicas: 1
  selector:
    matchLabels:
      app: skill-service
  template:
    metadata:
      labels:
        app: skill-service
    spec:
      containers:
      - name: skill-service
        image: skill-service
        ports:
        - containerPort: 8080
        envFrom:
        - configMapRef:
            name: skill-service-config
---
apiVersion: v1
kind: Service
metadata:
  name: skill-service
spec:
  selector:
    app: skill-service
  ports:
  - port: 8080
    targetPort: 8080
```

## ConfigMap for Environment Variables
```bash
# Generate ConfigMap from template
envsubst < k8s/configmap.yaml.template > k8s/configmap.yaml
kubectl apply -f k8s/configmap.yaml
```

## Kubernetes Secret for Service Account Keys
```bash
# Create key file
gcloud iam service-accounts keys create $SA.json \
  --iam-account $SA@$PROJECT_ID.iam.gserviceaccount.com

# Create K8s secret
kubectl create secret generic $SA-key \
  --from-file=$SA.json=$SA.json

# Mount in pod at /var/secrets/google/$SA.json
```

|---|---|---|---|
| Actual service instance | Medium | Low | High |

**GCP emulators available:**
