# Ch11 — Building the Citadel (Secure Production)

## Citadel Security Layers
```
Internet
  → Cloud Armor (WAF + DDoS protection)
  → Global HTTP(S) Load Balancer (TLS termination)
  → Cloud CDN (optional caching)
  → Cloud Run / GKE services (private, no-unauthenticated)
  → IAM / Workload Identity (service-to-service auth)
  → Secret Manager (credential management)
  → VPC + Private IP (data layer isolation)
```

## Global HTTP(S) Load Balancer
- Single anycast IP routed to nearest GCP PoP worldwide
- TLS termination at edge (not at service)
- Supports path-based routing to multiple backends
- Integrates with Cloud Armor, Cloud CDN, Identity-Aware Proxy

```bash
# Reserve global static IP
gcloud compute addresses create $IP_NAME --global

# Get the IP address
export IP=$(gcloud compute addresses describe $IP_NAME --global --format='value(address)')

# Create managed SSL certificate (auto-provisioned by Google)
gcloud compute ssl-certificates create $CERT_NAME \
  --domains=$DOMAIN --global

# Create serverless NEG (Network Endpoint Group) for Cloud Run
gcloud compute network-endpoint-groups create $NEG_NAME \
  --region=$REGION --network-endpoint-type=serverless \
  --cloud-run-service=$SERVICE_NAME

# Create backend service
gcloud compute backend-services create $BACKEND_NAME \
  --load-balancing-scheme=EXTERNAL_MANAGED --global

# Add NEG to backend
gcloud compute backend-services add-backend $BACKEND_NAME \
  --network-endpoint-group=$NEG_NAME \
  --network-endpoint-group-region=$REGION --global

# Create URL map (path routing)
gcloud compute url-maps create $URL_MAP_NAME \
  --default-service=$BACKEND_NAME

# Create HTTPS proxy
gcloud compute target-https-proxies create $PROXY_NAME \
  --url-map=$URL_MAP_NAME \
  --ssl-certificates=$CERT_NAME

# Create forwarding rule
gcloud compute forwarding-rules create $FWD_RULE_NAME \
  --address=$IP_NAME --global \
  --target-https-proxy=$PROXY_NAME --ports=443
```

## DNS Setup
```bash
# Create Cloud DNS managed zone
gcloud dns managed-zones create $ZONE_NAME \
  --dns-name=$DOMAIN --description="Skills Mapper DNS"

# Add A record
gcloud dns record-sets transaction start --zone=$ZONE_NAME
gcloud dns record-sets transaction add $IP \
  --name=$DOMAIN --ttl=300 --type=A --zone=$ZONE_NAME
gcloud dns record-sets transaction execute --zone=$ZONE_NAME

# Get nameservers (configure at registrar)
gcloud dns managed-zones describe $ZONE_NAME --format='value(nameServers)'
```

## Cloud Armor (WAF + DDoS)
```bash
# Create security policy
gcloud compute security-policies create $POLICY_NAME \
  --description="WAF policy"

# Add rate limiting rule
gcloud compute security-policies rules create 1000 \
  --security-policy=$POLICY_NAME \
  --action=rate-based-ban \
  --src-ip-ranges="*" \
  --rate-limit-threshold-count=100 \
  --rate-limit-threshold-interval-sec=60 \
  --ban-duration-sec=600

# Attach to backend service
gcloud compute backend-services update $BACKEND_NAME \
  --security-policy=$POLICY_NAME --global
```

## Identity-Aware Proxy (IAP)
- Protects Cloud Run/GCE/GKE services with Google identity
- Users must be authenticated (Google Account) to access
- Useful for internal tools without VPN

```bash
# Enable IAP on backend service
gcloud compute backend-services update $BACKEND_NAME \
  --iap=enabled --global

# Grant user access via IAP
gcloud projects add-iam-policy-binding $PROJECT_ID \
  --member=user:user@example.com \
  --role=roles/iap.httpsResourceAccessor
```

## Securing Cloud Run (No Public Access)
```bash
# Redeploy with no public access
gcloud run services update $SERVICE_NAME --no-allow-unauthenticated

# Allow only LB service account to invoke
gcloud run services add-iam-policy-binding $SERVICE_NAME \
  --member=serviceAccount:$LB_SA@$PROJECT_ID.iam.gserviceaccount.com \
  --role=roles/run.invoker
```

## Zero-Trust Service-to-Service Auth
```bash
# Service A calls Service B using its own identity token
TOKEN=$(gcloud auth print-identity-token --impersonate-service-account=$SA@$PROJECT_ID.iam.gserviceaccount.com)
curl -H "Authorization: Bearer $TOKEN" $SERVICE_B_URL

# In code (Go): use google.oauth2 service account credentials to fetch ID token
# Target audience = Service B URL
```

## Domain Mapping for Cloud Run (Simpler Alternative)
```bash
# Map custom domain directly to Cloud Run (no LB needed for simple cases)
gcloud run domain-mappings create --service=$SERVICE_NAME \
  --domain=$DOMAIN --region=$REGION

# Get DNS records to configure at registrar
gcloud run domain-mappings describe --domain=$DOMAIN --region=$REGION
```

## Security Checklist
- [ ] All services set to `--no-allow-unauthenticated`
- [ ] Dedicated service account per service (not default SA)
- [ ] Secrets in Secret Manager, not env vars or code
- [ ] HTTPS enforced (HTTP → HTTPS redirect)
- [ ] Cloud Armor WAF attached to public LB
- [ ] IAM conditions set (restrict time/resource where possible)
- [ ] VPC Service Controls for sensitive data
- [ ] Container vulnerability scanning enabled in Artifact Registry
