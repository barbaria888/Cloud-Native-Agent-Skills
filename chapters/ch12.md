# Ch12 — Building the Factory (CI/CD + IaC)

## Factory Purpose
Transform source code into a deployable immutable artifact and automate
delivery to the Citadel. Humans should not deploy manually.

## CI/CD Pipeline (Cloud Build)
```
Code Push (Git) → Cloud Build Trigger
  → Build container (Buildpacks/Jib/Docker)
  → Push to Artifact Registry
  → Run unit + integration tests
  → Deploy to staging (Cloud Run/GKE)
  → [Manual gate] → Deploy to production
```

## Cloud Build
```bash
gcloud services enable cloudbuild.googleapis.com

# Manually trigger a build
gcloud builds submit --tag $REGION-docker.pkg.dev/$PROJECT_ID/$REPO/$IMAGE .

# Trigger from push to main branch
gcloud builds triggers create github \
  --repo-name=$REPO \
  --repo-owner=$OWNER \
  --branch-pattern="^main$" \
  --build-config=cloudbuild.yaml
```

**cloudbuild.yaml:**
```yaml
steps:
  - name: 'gcr.io/google.com/cloudsdktool/cloud-sdk'
    entrypoint: gcloud
    args:
      - run
      - deploy
      - $SERVICE_NAME
      - --image
      - $REGION-docker.pkg.dev/$PROJECT_ID/$REPO/$IMAGE
      - --region=$REGION
  - name: 'gcr.io/cloud-builders/docker'
    args: ['build', '-t', '$_IMAGE', '.']
images:
  - '$_IMAGE'
```

## Artifact Registry Setup
```bash
gcloud services enable artifactregistry.googleapis.com

# Create Docker repository
gcloud artifacts repositories create $REPO_NAME \
  --repository-format=docker --location=$REGION

# Configure Docker to authenticate
gcloud auth configure-docker $REGION-docker.pkg.dev

# Vulnerability scanning is automatic for pushed images
```

## Terraform for Infrastructure as Code
```bash
# Install Terraform + initialize
terraform init

# Preview changes
terraform plan -var project_id=$PROJECT_ID

# Apply changes
terraform apply -var project_id=$PROJECT_ID

# Destroy (cleanup)
terraform destroy
```

**main.tf example (Cloud Function):**
```hcl
resource "google_cloudfunctions2_function" "tag_updater" {
  name     = "tag-updater"
  location = var.region

  build_config {
    runtime     = "go120"
    entry_point = "UpdateTags"
    source {
      storage_source {
        bucket = google_storage_bucket.source.name
        object = google_storage_bucket_object.source.name
      }
    }
  }

  service_config {
    max_instance_count = 1
    available_memory   = "256M"
    environment_variables = {
      BUCKET_NAME = google_storage_bucket.data.name
      OBJECT_NAME = "tags.csv"
    }
    service_account_email = google_service_account.tag_updater.email
  }
}
```

## Cloud Build Service Account Permissions
```bash
# Grant Cloud Build SA permission to deploy to Cloud Run
export CB_SA=$(gcloud projects describe $PROJECT_ID --format='value(projectNumber)')@cloudbuild.gserviceaccount.com

gcloud projects add-iam-policy-binding $PROJECT_ID \
  --member=serviceAccount:$CB_SA \
  --role=roles/run.admin

gcloud projects add-iam-policy-binding $PROJECT_ID \
  --member=serviceAccount:$CB_SA \
  --role=roles/iam.serviceAccountUser
```

## Continuous Delivery with Cloud Deploy
```bash
gcloud services enable clouddeploy.googleapis.com

# Define delivery pipeline
cat > pipeline.yaml << EOF
apiVersion: deploy.cloud.google.com/v1
kind: DeliveryPipeline
metadata:
  name: $PIPELINE_NAME
serialPipeline:
  stages:
  - targetId: staging
  - targetId: production
EOF

gcloud deploy apply --file=pipeline.yaml --region=$REGION
```

## Key Factory Principles
1. **Immutable artifacts**: the exact same image moves from test → staging → prod
2. **No manual deployments**: humans configure pipelines, not individual deploys
3. **Build once, deploy many**: build in CI, promote through environments
4. **Automated testing gates**: deployment blocked until all tests pass
5. **IaC for everything**: even the CI/CD pipeline is code (cloudbuild.yaml, Terraform)

## Container Signing / Attestation (Supply Chain Security)
```bash
# Binary Authorization: enforce only verified images can deploy to GKE
gcloud services enable binaryauthorization.googleapis.com

# Sign image after scanning
gcloud container binauthz attestors create $ATTESTOR_NAME \
  --attestation-authority-note=$NOTE_NAME \
  --attestation-authority-note-project=$PROJECT_ID
```
