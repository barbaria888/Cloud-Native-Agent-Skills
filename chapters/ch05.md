# Ch05 — Project 1: Tag Updater with Cloud Functions

## Architecture
```
Cloud Scheduler (weekly cron)
  → HTTP trigger
  → Cloud Function (Go) — reads BigQuery → writes to Cloud Storage
```

## Services Used
| Service | Role |
|---|---|
| BigQuery | Query Stack Overflow public dataset for tags |
| Cloud Storage | Store the resulting CSV list of tags |
| Cloud Functions Gen 2 | Run glue code serverlessly; HTTP + scheduler-triggered |
| Cloud Scheduler | Trigger function on weekly cron (0 0 * * SUN) |
| Artifact Registry | Store the auto-built container image (internal to CF) |
| Cloud Build | Build the container image (internal to CF) |

## Cloud Functions Key Facts
- Gen 2 is a wrapper around Cloud Run + Cloud Build
- Supports: Node.js, Python, Go, Java, Ruby, PHP, .NET Core
- Max memory: 16GiB | Max CPUs: 4 | Max execution time: 60 min
- Default: 256MB RAM, 1 vCPU, 60s timeout
- Code auto-packaged into managed container (container hidden from you)
- HTTP trigger registers entry point function

## BigQuery CLI Quick Reference
```bash
# Preview query results
bq query --max_rows=10 --nouse_legacy_sql --format=csv \
  "SELECT tag_name FROM bigquery-public-data.stackoverflow.tags ORDER BY tag_name"

# Export to file (always set max_rows to avoid surprise billing)
bq query --max_rows=100000 --nouse_legacy_sql --format=csv \
  "SELECT tag_name FROM bigquery-public-data.stackoverflow.tags ORDER BY tag_name" \
  > $FILE_NAME
```

**Billing tip:** BigQuery charges after 1TB/month scanned. Always specify `--max_rows`.

## Deploying Cloud Function
```bash
# Enable required services
gcloud services enable cloudfunctions.googleapis.com cloudbuild.googleapis.com \
  artifactregistry.googleapis.com run.googleapis.com

# Deploy
gcloud functions deploy $FUNCTION_NAME \
  --gen2 --runtime=go120 --region=$REGION \
  --trigger-http --no-allow-unauthenticated \
  --env-vars-file env.yaml

# Get URI
export FUNCTION_URI=$(gcloud functions describe $FUNCTION_NAME \
  --gen2 --format='value(serviceConfig.uri)')

# Test (authenticated)
curl -H "Authorization: Bearer $(gcloud auth print-identity-token)" $FUNCTION_URI

# Check logs
gcloud functions logs read $FUNCTION_NAME --gen2 --region=$REGION
# Stream from underlying Cloud Run service
gcloud beta run services logs tail $FUNCTION_NAME --project $PROJECT_ID
```

## Service Account Setup (Least Privilege)
```bash
# Create SA for function execution
gcloud iam service-accounts create $SA_NAME --display-name "tag-updater service account"

# BigQuery job permission
gcloud projects add-iam-policy-binding $PROJECT_ID \
  --member=serviceAccount:$SA_NAME@$PROJECT_ID.iam.gserviceaccount.com \
  --role=roles/bigquery.jobUser

# Cloud Storage object admin (create + delete)
gsutil iam ch serviceAccount:$SA_NAME@$PROJECT_ID.iam.gserviceaccount.com:objectAdmin \
  gs://$BUCKET_NAME

# Redeploy with service account
gcloud functions deploy $FUNCTION_NAME --gen2 --runtime=go120 \
  --service-account=$SA_NAME@$PROJECT_ID.iam.gserviceaccount.com \
  --trigger-http --no-allow-unauthenticated --env-vars-file env.yaml
```

## Cloud Scheduler Setup
```bash
gcloud services enable cloudscheduler.googleapis.com

# Create separate invoker service account
gcloud iam service-accounts create $INVOKER_SA_NAME

# Grant invoker role on the Cloud Run service underlying the function
gcloud run services add-iam-policy-binding $FUNCTION_NAME \
  --member=serviceAccount:$INVOKER_SA_NAME@$PROJECT_ID.iam.gserviceaccount.com \
  --role=roles/run.invoker

# Create weekly job
gcloud scheduler jobs create http $JOB_NAME \
  --schedule="0 0 * * SUN" \
  --uri=$FUNCTION_URI --max-retry-attempts=3 --location=$REGION \
  --oidc-service-account-email=$INVOKER_SA_NAME@$PROJECT_ID.iam.gserviceaccount.com \
  --oidc-token-audience=$FUNCTION_URI

# Manual trigger for testing
gcloud scheduler jobs run $JOB_NAME --location=$REGION
```

## Key Pattern: Two Service Accounts
1. **Function SA** — executes BigQuery queries + writes to Cloud Storage
2. **Invoker SA** — allowed to trigger the Cloud Run endpoint (given to Scheduler)

This pattern separates "execution" permissions from "invocation" permissions.
