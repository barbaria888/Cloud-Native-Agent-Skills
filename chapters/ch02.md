# Ch02 — Why Google Cloud Platform?

## GCP Differentiators
- Service integration (fewer services, deeper integration vs. AWS two-pizza teams)
- Open standards focus (OCI, Kubernetes, Knative, PostgreSQL wire protocol)
- Built by cloud-native engineers for cloud-native engineers

## Physical Infrastructure
```
Machine (10) → Rack → Row → Cluster → Data Center → Zone → Region (3-4 zones)
```
- All hardware standardized; any machine can run any server
- Proprietary cooling, power distribution, networking

## Core Software Abstractions (The Hidden Layer)

| Abstraction | What it manages | What it gives you |
|---|---|---|
| **Borg** | Compute scheduling | Declare CPU/RAM; Borg finds machines, heals failures |
| **Colossus** | Distributed storage | Declare bytes + IOPS; hot/cold tiering automatic |
| **Spanner** | Global database | Strong consistency across regions; TrueTime API |
| **Andromeda** | Network bandwidth | Software-defined; global LB and firewalls are API calls |
| **Jupiter** | Intra-DC network | High-speed fabric linking all machines in a datacenter |
| **B4** | WAN | Software-defined global network between datacenters |

## Why Not DIY?
- Treating GCP as IaaS (just VMs + disks) forfeits all higher-level abstractions
- Forces license costs + operational overhead (e.g. self-managed Kubernetes)
- "Kubernetes the Hard Way" illustrates the effort — managed GKE removes that burden
- Individual developers can't DIY; organisations shouldn't unless multi-cloud simultaneous deployment is required

## Vendor Lock-in Strategies
| Strategy | Portability | Managed Benefits |
|---|---|---|
| IaaS-only | High | Low |
| Kubernetes-based | Medium | Medium |
| **Open standards** (recommended) | **Medium-High** | **High** |
| Single-cloud | Low | Very High |

**Book recommendation:** prefer open standards (PostgreSQL, gRPC, Knative, OCI containers) — portable AND capable of using managed services.

## Key Insight
> GCP's real strength is the software Google built to manage machines — not the hardware.
> Using higher-level abstractions is what cloud-native development is about.
